## 1. 架构总览

    ├── fr.ts
    ├── pt.ts
    ├── de.ts

## 2. 核心 API

| API | 用途 | 调用方 |
|------|------|--------|
| `t(key, params?)` | 取翻译字符串。`params` 是 `{name}` 占位符的填充 map | 任何 UI 渲染处 |
| `setLocale(code)` | 切换 locale。相同 locale 是 no-op。会写 `<html lang>` + `dir` + 通知 subscribers | 用户在 settings 切语言（间接通过 `apply.ts`） |
| `initLocale(code)` | 启动时强制设置 locale（即使与默认值相同），触发 subscribers 第一次回调 | `initSettings()` 末尾 |
| `getLocale()` | 当前 locale code | 调试、调试日志 |
| `resolveLocale(pref)` | 把 `'auto' / 'zh-CN' / 'ja' / 'fr-CA' / 'invalid'` 解析为 `LocaleCode` | i18n 模块 + background SW |
| `subscribe(cb)` | 订阅 `setLocale` 通知，返回 unsubscribe 函数 | `applyLocaleToDom` 等需要在切语言时重画 UI 的模块 |
| `listLocales()` | 返回所有已 catalog 的 `LocaleBundle[]`（含 selfName / englishName） | settings 面板的语言下拉 |

### 2.1 `t()` 的 fallback 链
1. 当前 locale 的 catalog 里有这个 key → 用它
2. 没有 → fallback 到 `en` 翻译
3. `en` 也没有（理论不可能 —— tsc 强制）→ 返回原始 key id（界面至少 grep-able）

### 2.2 `resolveLocale()` 的解析算法
```
input: 'auto' (or null)
  ↓ read navigator.language
input: 'zh-CN'
  ↓ exact match in SUPPORTED_LOCALES? 'zh-CN' ∉ list → no
  ↓ primary subtag: 'zh' ∈ list → return 'zh'

input: 'sv-SE'
  ↓ exact match? no
  ↓ primary subtag: 'sv' ∉ list → no
  → return 'en'  // fallback
```

`SUPPORTED_LOCALES = ['en','zh','es','ar','hi','fr','pt','de','ja','ru']`。要加新 locale（如 `it`）必须先在 types.ts 加进 SUPPORTED_LOCALES + RTL_LOCALES（如适用）+ catalog 目录加文件 + i18n/index.ts 导入 + CATALOG 列表 + background SW 的 SETTINGS_MENU_TITLES + 设置面板的 languages select。

## 3. 修改流程

### 3.1 新增一个 key

例：新增「重置列宽」按钮。

**Step 1**：在 `types.ts` 的 `MessageKey` 联合加 key（按字母序 / 按使用区域插入都可以，但保持一致）：
```ts
// types.ts
export type MessageKey =
  | 'settings.field.resetColumnWidth'
  | 'settings.field.resetColumnWidthDesc'
  // ...
  ;
```

**Step 2**：`en.ts` 加翻译：
```ts
'settings.field.resetColumnWidth': 'Reset column width',
'settings.field.resetColumnWidthDesc': 'Restore the column width to its default value.',
```

**Step 3**：9 个其它 catalog 同步加 —— tsc 编译期强制要求。

**Step 4**：在调用方用 `t()`：
```ts
btn.textContent = t('settings.field.resetColumnWidth');
descEl.textContent = t('settings.field.resetColumnWidthDesc');
```

**Step 5**：`pnpm exec tsc --noEmit` 确认 0 error。

### 3.2 修改一个已有 key 的文案

直接编辑 10 个 catalog 的字符串。**不要**改 key 本身（改 key 会破坏 MessageKey 联合 + tsc 立刻报）。

### 3.3 静态配置对象里加字符串

错误（裸字符串）❌：
```ts
const MODE_OPTIONS = [
  { value: 'light', title: '亮' },  // 永远都是「亮」，切语言也没用
  { value: 'system', title: '跟随系统' },
  { value: 'dark', title: '暗' },
];
```

正确（用 `MessageKey` 字段）✅：
```ts
import { t, type MessageKey } from '../lib/i18n';

type Mode = 'light' | 'system' | 'dark';
const MODE_OPTIONS: ReadonlyArray<{ value: Mode; titleKey: MessageKey }> = [
  { value: 'light', titleKey: 'appearanceToggle.light' },
  { value: 'system', titleKey: 'appearanceToggle.system' },
  { value: 'dark', titleKey: 'appearanceToggle.dark' },
];

// 渲染时：
btn.title = t(opt.titleKey);
```

参考实现：
- [`src/features/bookmarks/folder-actions.ts`](../src/features/bookmarks/folder-actions.ts)（`ActionButton.titleKey`）
- [`src/newtab/appearance-toggle.ts`](../src/newtab/appearance-toggle.ts)（`Option.titleKey`）
- [`src/popup/layout-picker.ts`](../src/popup/layout-picker.ts)（`LAYOUTS[].labelKey`）

### 3.4 插值字符串

支持 `{name}` 占位符：
```ts
t('folderAction.confirmOpenAll', { count: urlCount })
// en: "Open all {count} links in this folder?" → "Open all 8 links in this folder?"
// zh: "打开此文件夹中全部 {count} 个链接？" → "打开此文件夹中全部 8 个链接？"
```

参数只支持 `string | number`。`null` / `undefined` / `boolean` 会先 `String(v)` 转换。

**禁止**：用模板字符串拼接（`"打开" + count + "个"`）—— 绕过 catalog。

## 4. live refresh（不刷新页面）

切语言时整个 UI 自动 in-place 重画，不需要刷新页面。链路：

```
用户切语言（settings panel 语言下拉）
  → chrome.storage.sync.settings.language 写
  → apply.ts 监听 chrome.storage.onChanged
  → setLocale(newCode) 写 <html lang/dir>
  → i18n.subscribe 通知
  → applyLocaleToDom() (newtab/main.ts)
    → updateTopbarStrings()        # search placeholder, settings button title
    → updateAppearanceToggleStrings()  # 暗色模式按钮 tooltip
    → updateUndoStrings()           # undo button label / count
    → updateSearchStrings()         # 搜索框 footer hints（in-place 替换）
    → void renderColumns()          # 特殊目录 title 重新走 t()
    → refreshSettingsPanelLocale()  # 打开中的 settings panel 在原位重画
  → background SW 也监听 chrome.storage.onChanged
  → removeAll() + 重新 chrome.contextMenus.create()
```

**关键**：settings panel **必须**走 `refreshSettingsPanelLocale()`（in-place 重画），**不要**走 `closeSettingsPanel() + openSettingsPanel()`（会闪动，参见 v0.2.121 修复）。

## 6. background SW 的独立表

`src/background.ts` 是独立的 service worker build chunk。**不能** import `src/lib/i18n/` —— 那个模块会拉 chrome.bookmarks shims 进 SW chunk，超出 10KB 预算。

替代方案：维护一个**最小**的 `SETTINGS_MENU_TITLES: Record<string, string>` 表，存放 background SW 自己用的 1 个字符串（右键菜单「Open Settings」）。当前 10 个 locale 全填。

**何时同步**：
- 新加 locale（`it`）→ 在 `SETTINGS_MENU_TITLES` 加 `'it': 'Apri impostazioni'`
- 改文案 → 同步 10 个值

**自检**：
```bash
# SW 表里少了某 locale，chrome.storage.onChanged 不会报错，菜单会 fallback 到 'en'
# 但加新 locale 时 tsc 不会强制 —— 类型是 Record<string, string>，key 是 string
# 所以 PR review 必须人工对照 en.ts
grep "actionMenu.openSettings" src/lib/i18n/catalog/*.ts
grep "SETTINGS_MENU_TITLES" src/background.ts
```

## 7. RTL 适配（Arabic `ar`）

`setLocale('ar')` 自动写 `<html dir="rtl">`。项目 CSS 走 `flex + gap`（`CLAUDE.md §9.2`），多数 UI 在 RTL 下自动镜像 —— 不需要任何 per-component 改动。

需要显式处理的：
- **URL / hostname 显示节点**：Arabic 文字中夹杂英文 host name 时，断行可能错乱。强制 LTR + `unicode-bidi: isolate`：
  ```css
  [dir="rtl"] .search-result-url,
  [dir="rtl"] .split-picker-url,
  [dir="rtl"] .picker-url,
  [dir="rtl"] .bookmark-url,
  [dir="rtl"] .search-result-host,
  [dir="rtl"] .sp-pill-url {
    direction: ltr;
    unicode-bidi: isolate;
  }
  ```
- **search input**：用户输入方向 = UI 方向
  ```css
  [dir="rtl"] .search-input {
    direction: rtl;
    text-align: right;
  }
  ```
- **column 排布**：`flex-direction: row`（项目规范）自动镜像 —— 列从右到左排列（符合 RTL 期望）
- **不要**手动加 `flex-direction: row-reverse` 来"镜像" —— `<html dir="rtl">` 已经处理

新增 URL 显示节点（如新浮层）→ 把 selector 加进 [`styles/globals.css`](../styles/globals.css) 末尾的 `[dir="rtl"]` 列表。

## 8. 翻译校对流程

### 8.1 用户手动微调

直接编辑 `src/lib/i18n/catalog/<code>.ts` 里的字符串值。**不要**改 key 名字。

```ts
// 错误
'folderAction.openAll': '打开全部链接',  // ← 改 key

// 正确
'folderAction.openAll': '打开文件夹内全部链接',  // ← 改值
```

### 8.2 加新 locale

1. 在 `types.ts` 加 `LocaleCode`（如 `'it'`）
2. 在 `SUPPORTED_LOCALES` 加 `'it'`
3. 在 `RTL_LOCALES` 加 `'it'`（如适用，Hebrew / Persian / Urdu 等）
4. 在 `catalog/` 加 `it.ts` 文件，对齐 `en.ts` 全部 ~210 keys
5. 在 `index.ts` `CATALOG` 列表加 `it`
6. 在 background `SETTINGS_MENU_TITLES` 加 `'it': '...'`
7. 在 settings panel 的 language select 选项加（`settings-panel.ts:createLanguageSelect()`）
8. `pnpm exec tsc --noEmit` 0 error
9. `pnpm build` 0 error
10. 手动 reload 扩展，切到新 locale 测一遍 §6.7 checklist

### 8.3 AI 辅助翻译

当用户用 AI 工具（ChatGPT、Claude）批量生成新 locale 时：
- 给 AI 一份 `en.ts` + 一个输出模板（schema + catalog 类型）
- 强制 AI 保留所有 key 名字 + 顺序
- AI 输出后**人工 review hi / ar / ru / ja**（高错率语言）
- 校对完后 `pnpm exec tsc --noEmit` 确认通过

## 9. 常见错误模式

| 症状 | 原因 | 修复 |
|------|------|------|
| `tsc` 报 `'foo.bar' is not assignable to MessageKey` | catalog 里写错了 key | 改 key 名字匹配 types.ts |
| `tsc` 报 `Property 'foo' is missing in type` | 某个 catalog 漏了一个 key | 在那个 catalog 加 key |
| 切语言后某处没变 | 调用方用了 `t()` 但没在 `applyLocaleToDom()` 触发重画 | export `updateXxxStrings()` 并接入 `applyLocaleToDom()` |
| 切语言后 settings panel 闪一下 | 调了 `closeSettingsPanel() + openSettingsPanel()` | 改用 `refreshSettingsPanelLocale()` |
| 切到 Arabic 时 URL 跳行 | 新加的 URL 元素没在 `[dir="rtl"]` 列表 | 把 selector 加进 globals.css |
| Label 太长被截断 | `.sp-label` 被加了 `nowrap` / `ellipsis` | 改回 v0.2.121 后的 `white-space: normal; word-break: break-word` |
| 浏览器 action 右键「打开设置」没切 | `SETTINGS_MENU_TITLES` 漏了某个 locale | 在 background.ts 加该 locale |
| 切语言后 popup 还是旧文案 | popup `app.ts` 顶部没 `await initSettings()` | 在 `render(root)` 之前 `await initSettings()` |
